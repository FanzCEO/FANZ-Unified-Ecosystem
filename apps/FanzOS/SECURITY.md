Reporting Process

Submit Report: Email detailed vulnerability information to security@fanz.foundation
Response Time: Initial acknowledgment within 48 hours
Assessment: Vulnerability assessment completed within 5-7 business days
Updates: Progress updates provided every 3-5 days during investigation
What to Include

Detailed description of the vulnerability
Steps to reproduce the issue
Potential impact assessment
Any proof-of-concept code (if applicable)
Your contact information for follow-up
Response Timeline

Critical: Immediate response (within 24 hours)
High: Response within 48 hours
Medium: Response within 5 business days
Low: Response within 10 business days
What to Expect

If Accepted:

Acknowledgment and credit (unless you prefer anonymity)
Timeline for fix development and deployment
Notification when fix is released
Optional inclusion in our security acknowledgments
If Declined:

Detailed explanation of why the issue doesn't qualify as a security vulnerability
Alternative suggestions if applicable
Guidance on proper channels for feature requests or bug reports
Security Best Practices for Contributors

Development

All code must pass security linting and audit checks
Dependencies must be regularly updated and scanned
Input validation required for all user-facing endpoints
Secure coding practices mandatory
Deployment

Use only the provided GitHub Actions workflows
Ensure self-hosted runner security configurations
Regular security updates and patches
Monitor for unusual activity
Responsible Disclosure

We appreciate security researchers who help keep Fanz Operating System secure. We commit to:

Working with you to understand and resolve issues quickly
Keeping you informed throughout the resolution process
Recognizing your contribution (if desired)
Not taking legal action against researchers who follow responsible disclosure
